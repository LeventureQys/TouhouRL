# 动作空间设计详解

## 按键系统

东方Project的操作按键：
- **方向键**：上、下、左、右（4个独立按键）
- **Shift键**：低速移动（精密躲避）
- **Z键**：射击
- **X键**：Bomb（清屏+无敌）

## 按帧执行的动作表示

在强化学习中，每一帧都需要输出一个动作。由于是按帧执行，动作应该表示为**当前帧所有按键的状态**。

### 方案A：多二元动作（Multi-Binary Action）

**推荐用于：** 最灵活的控制，适合高级训练

**动作表示**：
```python
action = [up, down, left, right, shift, bomb]
# 每个元素为 0（未按下）或 1（按下）
```

**示例**：
```python
[1, 0, 0, 0, 0, 0]  # 按上
[0, 0, 1, 0, 0, 0]  # 按左
[1, 0, 1, 0, 0, 0]  # 按左上
[1, 0, 1, 0, 1, 0]  # 按左上+低速
[0, 0, 0, 0, 0, 1]  # 使用Bomb
```

**特点**：
- 动作空间：6维二元向量
- 理论组合数：2^6 = 64种
- 实际有效组合：约30-40种（排除上下同时按、左右同时按等无效组合）
- 优点：最灵活，可以表达任意按键组合
- 缺点：动作空间较大，需要更多训练

**实现**：
```python
import gym
from gym import spaces

class TouhouEnv(gym.Env):
    def __init__(self):
        # 动作空间：6个二元动作
        self.action_space = spaces.MultiBinary(6)
        # [up, down, left, right, shift, bomb]

    def step(self, action):
        # action是一个长度为6的数组
        up, down, left, right, shift, bomb = action

        # 执行按键
        if up: press_key('UP')
        if down: press_key('DOWN')
        if left: press_key('LEFT')
        if right: press_key('RIGHT')
        if shift: press_key('SHIFT')
        if bomb: press_key('X')

        # ... 游戏逻辑
```

### 方案B：离散动作组合（Discrete Action）

**推荐用于：** 简化训练，适合初期快速验证

**动作表示**：
```python
# 方向动作（9种）
directions = {
    0: [],              # 停止
    1: ['UP'],          # 上
    2: ['DOWN'],        # 下
    3: ['LEFT'],        # 左
    4: ['RIGHT'],       # 右
    5: ['UP', 'LEFT'],  # 左上
    6: ['UP', 'RIGHT'], # 右上
    7: ['DOWN', 'LEFT'],# 左下
    8: ['DOWN', 'RIGHT']# 右下
}

# Shift状态（2种）
shift_states = {
    0: False,  # 高速
    1: True    # 低速
}

# Bomb状态（2种）
bomb_states = {
    0: False,  # 不使用
    1: True    # 使用
}

# 总动作数：9 × 2 × 2 = 36种
```

**实现**：
```python
class TouhouEnv(gym.Env):
    def __init__(self):
        # 动作空间：36个离散动作
        self.action_space = spaces.Discrete(36)

    def step(self, action):
        # 解码动作
        direction_idx = action // 4
        shift_bomb = action % 4
        shift = shift_bomb // 2
        bomb = shift_bomb % 2

        # 执行方向键
        direction_keys = directions[direction_idx]
        for key in direction_keys:
            press_key(key)

        # 执行Shift
        if shift:
            press_key('SHIFT')

        # 执行Bomb
        if bomb:
            press_key('X')
```

**特点**：
- 动作空间：36个离散动作
- 优点：动作空间较小，训练更快
- 缺点：无法表达某些特殊组合（如只按Shift不移动）

### 方案C：简化动作（Simplified Action）

**推荐用于：** 最初期的概念验证

**动作表示**：
```python
# 仅9个方向动作
actions = {
    0: [],              # 停止
    1: ['UP'],          # 上
    2: ['DOWN'],        # 下
    3: ['LEFT'],        # 左
    4: ['RIGHT'],       # 右
    5: ['UP', 'LEFT'],  # 左上
    6: ['UP', 'RIGHT'], # 右上
    7: ['DOWN', 'LEFT'],# 左下
    8: ['DOWN', 'RIGHT']# 右下
}
```

**简化规则**：
- 自动保持射击（Z键始终按下）
- 自动速度切换：
  - 子弹密集时自动切换低速
  - 安全时自动切换高速
- Bomb由规则触发：
  - 生命值低且危险时自动使用
  - 或完全不使用Bomb

**实现**：
```python
class TouhouEnv(gym.Env):
    def __init__(self):
        self.action_space = spaces.Discrete(9)

    def step(self, action):
        # 执行方向
        direction_keys = actions[action]
        for key in direction_keys:
            press_key(key)

        # 自动射击
        press_key('Z')

        # 自动速度切换
        if self.is_dangerous():
            press_key('SHIFT')

        # 自动Bomb（可选）
        if self.should_use_bomb():
            press_key('X')
```

**特点**：
- 动作空间：9个离散动作
- 优点：最简单，训练最快
- 缺点：缺乏灵活性，性能上限较低

## 动作空间选择建议

### 训练阶段建议

**阶段1：概念验证（1-2周）**
- 使用方案C（简化动作）
- 目标：验证环境搭建、视觉系统、基础训练流程
- 在简单场景中测试

**阶段2：基础训练（2-4周）**
- 使用方案B（离散动作组合）
- 目标：学习基本的躲避和攻击
- 加入Shift和Bomb控制

**阶段3：高级训练（4周+）**
- 使用方案A（多二元动作）
- 目标：学习复杂操作，追求高分
- 完整游戏训练

### 不同算法的适配

**DQN系列**：
- 适合方案B和C（离散动作）
- 如果使用方案A，需要为每个按键训练独立的Q网络

**PPO/A3C**：
- 适合所有方案
- 方案A可以使用多个独立的Bernoulli分布
- 方案B/C使用Categorical分布

**SAC/TD3**：
- 适合方案A（连续动作的变体）
- 可以将动作表示为连续值，然后二值化

## 动作约束与后处理

### 无效动作过滤

某些动作组合是无效的，应该在执行前过滤：

```python
def filter_invalid_action(action):
    """过滤无效的动作组合"""
    up, down, left, right, shift, bomb = action

    # 上下不能同时按
    if up and down:
        up, down = 0, 0

    # 左右不能同时按
    if left and right:
        left, right = 0, 0

    return [up, down, left, right, shift, bomb]
```

### Bomb冷却

Bomb使用后有冷却时间，需要在环境中维护：

```python
class TouhouEnv(gym.Env):
    def __init__(self):
        self.bomb_cooldown = 0

    def step(self, action):
        # ... 其他逻辑

        # Bomb冷却
        if self.bomb_cooldown > 0:
            self.bomb_cooldown -= 1
            action[5] = 0  # 强制不使用Bomb

        # 如果使用了Bomb
        if action[5] == 1:
            self.bomb_cooldown = 60  # 60帧冷却
```

### 动作平滑

为了避免抖动，可以对动作进行平滑：

```python
class ActionSmoother:
    def __init__(self, window_size=3):
        self.window_size = window_size
        self.history = []

    def smooth(self, action):
        self.history.append(action)
        if len(self.history) > self.window_size:
            self.history.pop(0)

        # 多数投票
        smoothed = []
        for i in range(len(action)):
            votes = sum(a[i] for a in self.history)
            smoothed.append(1 if votes > len(self.history) / 2 else 0)

        return smoothed
```

## 实现示例

### 完整的多二元动作环境

```python
import gym
from gym import spaces
import numpy as np

class TouhouEnv(gym.Env):
    def __init__(self):
        super().__init__()

        # 动作空间：6个二元动作
        # [up, down, left, right, shift, bomb]
        self.action_space = spaces.MultiBinary(6)

        # 状态空间（示例）
        self.observation_space = spaces.Box(
            low=0, high=255,
            shape=(84, 84, 4),  # 4帧灰度图
            dtype=np.uint8
        )

        self.bomb_cooldown = 0
        self.bomb_count = 3

    def step(self, action):
        # 过滤无效动作
        action = self._filter_action(action)

        # 执行动作
        self._execute_action(action)

        # 获取新状态
        observation = self._get_observation()

        # 计算奖励
        reward = self._calculate_reward()

        # 检查是否结束
        done = self._is_done()

        # 额外信息
        info = {
            'lives': self.lives,
            'bombs': self.bomb_count,
            'score': self.score
        }

        return observation, reward, done, info

    def _filter_action(self, action):
        """过滤无效动作"""
        action = action.copy()
        up, down, left, right, shift, bomb = action

        # 上下互斥
        if up and down:
            action[0], action[1] = 0, 0

        # 左右互斥
        if left and right:
            action[2], action[3] = 0, 0

        # Bomb冷却
        if self.bomb_cooldown > 0:
            action[5] = 0
            self.bomb_cooldown -= 1

        # Bomb数量检查
        if self.bomb_count <= 0:
            action[5] = 0

        return action

    def _execute_action(self, action):
        """执行动作"""
        up, down, left, right, shift, bomb = action

        # 释放所有按键
        release_all_keys()

        # 按下对应按键
        if up: press_key('UP')
        if down: press_key('DOWN')
        if left: press_key('LEFT')
        if right: press_key('RIGHT')
        if shift: press_key('SHIFT')

        # 始终射击
        press_key('Z')

        # Bomb
        if bomb:
            press_key('X')
            self.bomb_cooldown = 60
            self.bomb_count -= 1

        # 等待一帧
        time.sleep(1/60)

    def reset(self):
        # 重置游戏
        self.bomb_cooldown = 0
        self.bomb_count = 3
        self.lives = 3
        self.score = 0

        # 返回初始状态
        return self._get_observation()
```

## 总结

动作空间设计需要在**灵活性**和**训练效率**之间权衡：

- **方案A（多二元）**：最灵活，但训练最慢
- **方案B（离散组合）**：平衡方案
- **方案C（简化）**：最快，但性能上限低

建议采用**渐进式策略**：从简单开始，逐步增加复杂度。
